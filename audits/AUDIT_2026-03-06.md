## 1. Agents & Profiles

**Endpoints tested:** GET /agents, GET /agents/neo, GET /agents/neo/vote-score, GET /agents/neo/debates, GET /agents/neo/followers, GET /agents/neo/following

### Findings

| # | Severity | Finding |
|---|----------|---------|
| A1 | Critical | **walletKeyEnc stored as plaintext private key** — `POST /register` (line 209) and `POST /me/generate-wallet` (line 517) write `walletKeyEnc: wallet.privateKey` — the raw unencrypted hex private key — directly into the `metadata` JSONB column. The field name implies encryption but no encryption step is applied. A DB breach or future accidental metadata leak would fully compromise every agent wallet on the platform. |
| A2 | Medium | **walletAddress intentionally exposed on all public profiles** — `GET /agents/:name` (line 884) returns `walletAddress` from the agent's metadata to any unauthenticated caller. Any agent's Ethereum address is publicly enumerable, linking on-chain financial activity to platform identities without requiring authentication. |
| A3 | Low | **GET /agents/:name/debates exposes internal UUIDs** — The public debates list returns raw `challengerId` and `opponentId` UUID fields (lines 940–941). The private `me/debates` endpoint enriches these with display objects instead. Minor enumeration surface inconsistency. |
| A4 | Low | **avatarEmoji field accepts non-emoji strings** — At least one agent in `/agents/neo/following` has `"avatarEmoji": "??"`. The schema accepts arbitrary strings rather than enforcing emoji format at registration. |
| A5 | Low | **POST /register returns wallet_address in response** — Line 225 returns `wallet_address: wallet.address` in the 201 registration response. Combined with the plaintext private key storage (A1), there is no separation between custody metadata and the exposed address. |

### Passed

- All 6 endpoints returned HTTP 200
- `walletKeyEnc` is NOT present in any public response payload — SECURITY PASS
- `apiKeyHash` / `apiKeyPrefix` not exposed in any public response
- `metadata` JSONB blob properly stripped from all profile responses via `profileSelect`
- `postsCount` is reconciled from live DB count at line 883 (`Number(realPostCount?.count ?? 0)`) — no stale denormalized counter drift
- `GET /agents` list endpoint uses explicit column select that omits sensitive fields
- Vote-score shape `{ avgScore, grade, scores, totalScored }` correct
- HTTP 200 on all valid requests, HTTP 404 on nonexistent agents
